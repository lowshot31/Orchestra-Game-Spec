# 💻 Tutti CLI 완벽 사용 설명서

환영합니다! Tutti CLI는 디스코드 봇을 세팅할 필요 없이, **터미널(명령창) 환경에서 가장 빠르고 가볍게 멀티 에이전트 협업을 실행하고 테스트할 수 있는 도구**입니다.

이 문서는 개발자나 시스템 검증을 원하는 평가자를 위해 작성되었습니다.

---

## 1. 🚀 왜 CLI 모드를 쓰나요?

- **Zero Setup:** 디스코드 개발자 포털 가입, 토큰 발급 등의 과정이 전혀 필요 없습니다.
- **빠른 검증:** `.env` 파일 설정 없이도 즉시 에이전트 파이프라인의 엔드투엔드(E2E) 동작을 확인할 수 있습니다.
- **CI/CD 연동:** 쉘 스크립트나 자동화 파이프라인에 쉽게 통합할 수 있습니다.

---

## 2. ⚡ 빠른 시작 (Mock 모드)

가장 기본적이고 빠른 방법입니다. 외부 API나 LLM 서버 없이, 내장된 템플릿 로직으로 동작합니다.

```bash
# 가상환경 활성화 후
python -m orchestra.cli
```

실행 시 `Designer`, `Reviewer`, `CEO` 에이전트가 가상의 아이디어를 바탕으로 기획안을 주고받는 과정을 터미널에서 즉시 볼 수 있습니다.

---

## 3. 🚦 개입 테스트 (Human-in-the-Loop)

CLI에서도 디스코드의 **[✅ 승인] / [✏️ 수정]** 버튼과 동일한 수준의 실시간 개입이 가능합니다.

명령어를 실행하면 1라운드(초안 작성 및 리뷰)가 끝난 후 다음과 같이 화면이 잠시 멈춥니다.

```text
============================================================
  Round 1 - Design Review
============================================================
...
[Risk] CEO Risk Assessment: MEDIUM
------------------------------------------------------------
에이전트 리뷰를 확인했습니다.
개입하려면 지시를 입력하세요 (Enter로 스킵):
> _
```

이때 두 가지 선택이 가능합니다.

1. **승인 (그냥 넘어가기):** 키보드의 `Enter`를 치면 즉시 최종 명세서 작성 및 코딩 단계로 넘어갑니다.
2. **방향성 수정 (개입):** 입력창에 지시사항을 적고 `Enter`를 칩니다.
   - 예: `장애물을 피하는 것 말고, 타이밍에 맞춰 버튼을 누르는 리듬 게임으로 바꿔줘`
   - *참고: 개입된 피드백은 2차 기획안(Revision)을 만들기 위해 무조건 **Designer**에게 전달됩니다.*

> 💡 **왜 CLI에서는 `@CEO` 같은 자유 채팅이 안 되나요?**
> Claude Code나 일반적인 AI CLI 도구들은 **대화형(REPL) 인터페이스**를 제공하지만, Tutti CLI는 **"End-to-End 파이프라인 자동화"**에 초점을 맞춘 1회성 실행 스크립트(Runner)로 구현되어 있습니다. 따라서 터미널에서는 복잡한 분기 없이 정해진 흐름(초안 ➡️ 리뷰 ➡️ 개입 ➡️ 수정 ➡️ 최종 산출물)을 빠르게 뽑아내는 데 사용되며, 에이전트들과의 자유로운 티키타카(채팅) 기능은 협업 툴인 **디스코드 봇**에 집중하여 구현했습니다.

## 4. 🎛️ 고급 명령어 옵션

CLI는 다양한 인자(arguments)를 지원하여 원하는 환경으로 커스텀 실행이 가능합니다.

### 💡 특정 아이디어로 시작하기
```bash
python -m orchestra.cli --idea "1분 안에 끝나는 모바일 수박게임"
```

### 🤖 실제 AI 모델 연동하기 (Ollama / API)
```bash
# 로컬 Ollama 모델 사용 (.env 설정 필요)
python -m orchestra.cli --mode ollama --idea "우주 비행 슈팅 게임"

# OpenAI / Gemini 외부 API 사용 (.env 설정 필요)
python -m orchestra.cli --mode api --idea "고양이 키우기 방치형 게임"
```

### ⚙️ 성향(Preset) 바꾸기
에이전트의 기획 성향을 조절할 수 있습니다. (`fast_draft`, `balanced`, `deep_review` 중 택 1)
```bash
python -m orchestra.cli --preset deep_review --idea "복잡한 경제 시스템을 가진 RPG"
```

### 🔗 실행과 동시에 GitHub PR 생성하기
기획이 완료되자마자 내 GitHub 저장소로 자동 PR을 생성할 수 있습니다.
```bash
python -m orchestra.cli --idea "방치형 게임" --github-repo "내아이디/레포이름" --github-token "ghp_나의토큰"
```

## 5. 📁 생성된 결과물 확인

명령어 실행이 완전히 끝나면 터미널 마지막에 결과물이 저장된 폴더 경로가 출력됩니다.
기본적으로 `artifacts/cli/playable/<run-name>` 폴더 안에 다음 파일들이 생성됩니다.

- `round_1_design.md` (1차 기획안)
- `round_2_revision.md` (수정 기획안)
- `final_game_spec.md` (최종 게임 명세서)
- `game_schema.json` (머신 리더블 데이터 스키마)
- `game/index.html` (브라우저에서 클릭하면 즉시 실행되는 게임 프로토타입!)

생성된 `index.html`을 크롬 등 웹 브라우저로 드래그해서 기획된 게임을 플레이해 보세요!
