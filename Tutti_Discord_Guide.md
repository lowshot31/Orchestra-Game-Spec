# 🎮 Tutti: 멀티 에이전트 게임 기획 봇 완벽 사용 설명서

환영합니다! Tutti(Orchestra)는 여러분의 머릿속에 있는 게임 아이디어를 **실제 기획서와 플레이 가능한 프로토타입**으로 만들어주고, 이를 **개발팀에게 GitHub PR로 바로 넘겨주는(Handoff)** 지능형 디스코드 봇입니다.

이 문서는 프로그래밍이나 GitHub를 전혀 모르는 분들도 쉽게 따라 할 수 있도록 작성된 **초보자용 완벽 가이드**입니다.

---

## 1. 🚀 시작하기 전 필수 세팅 (딱 한 번만 하시면 됩니다!)

Tutti가 여러분을 대신해 기획서를 작성하고 GitHub에 코드를 올리려면 권한(API 키)이 필요합니다.

### 🔑 1단계: GitHub 토큰(PAT) 발급받기

Tutti가 내 GitHub에 대신 PR을 올려주기 위한 "출입증"을 발급받는 과정입니다.

1. 브라우저에서 [GitHub Developer Settings](https://github.com/settings/tokens/new) 페이지에 접속합니다. (로그인 필요)
2. **Note** 칸에 `tutti-bot` 이라고 적습니다.
3. 스크롤을 살짝 내려서 **Select scopes** 항목 중 제일 위에 있는 **`[v] repo`** 체크박스 딱 하나만 체크합니다! ✅
4. 스크롤을 맨 밑으로 내려 초록색 **[Generate token]** 버튼을 누릅니다.
5. 화면에 `ghp_` 로 시작하는 긴 비밀번호가 나옵니다. 이걸 복사해 둡니다! (이 창을 닫으면 다시 볼 수 없으니 꼭 복사하세요)

### 📁 2단계: 결과물을 저장할 GitHub 레포지토리 만들기

1. GitHub 우측 상단의 `+` 버튼을 누르고 **New repository**를 클릭합니다.
2. **Repository name**에 `tutti-game-artifacts` 라고 적습니다.
3. ⚠️ 중요: **[v] Add a README file** 체크박스를 꼭! 체크해 주세요.
4. **[Create repository]** 버튼을 누릅니다.

### 🤖 3단계: 디스코드 봇에 정보 입력하기

이제 디스코드 채널로 돌아와서 봇에게 방금 만든 정보를 알려줍니다.

**A. 토큰 등록하기**

1. 채팅창에 `/tutti apikey` 를 치고 엔터를 누릅니다.
2. 봇이 DM(개인 메시지)으로 초록색 **[GitHub Token]** 버튼을 보내줍니다.
3. 버튼을 누르고:
   - Key Name 란에: `GITHUB_TOKEN` 이라고 적습니다. (대문자)
   - API Key 란에: 아까 복사해둔 `ghp_...` 토큰을 붙여넣습니다.
4. 제출 버튼을 누릅니다.

**B. 레포지토리 주소 등록하기**

1. 채팅창에 `/tutti settings` 를 치고 엔터를 누릅니다.
2. 파란색 **[🔧 Server]** 버튼을 누릅니다.
3. **GitHub Game Repo** 칸에 방금 만든 레포지토리 주소를 적습니다.
   - ❌ 잘못된 예: `https://github.com/내아이디/tutti-game-artifacts`
   - ✅ 올바른 예: `내아이디/tutti-game-artifacts` (전체 링크를 붙여넣어도 봇이 똑똑하게 잘라주긴 합니다!)
4. 제출을 누릅니다.

🎉 **축하합니다! 이제 모든 세팅이 끝났습니다.** 🎉

## 2. 🎮 실전! 게임 기획부터 배포까지 (Workflow)

세팅이 끝났으니 이제 봇을 부려먹을 시간입니다.

### 📝 Step 1: 기획 시작하기

디스코드 채팅창에 아래와 같이 명령어를 칩니다.

> `/tutti start name:dino-game idea:1분 안에 끝나는 우주선 피하기 게임 만들어줘`

_(주의: `name`에는 띄어쓰기 없는 영문/숫자 프로젝트 이름을, `idea`에는 구체적인 아이디어를 적어주세요!)_

### 🗣️ Step 2: AI 에이전트들의 브레인스토밍 (쓰레드 대화)

명령어를 치면 봇이 새로운 **쓰레드(Thread)**를 만듭니다.
쓰레드에 들어가 보면 3명의 AI가 열띤 회의를 합니다.

- 🎨 **디자이너:** "우주선 피하기 게임? 장애물을 소행성으로 하고 속도가 점점 빨라지게 하자!"
- 🛠️ **리뷰어:** "1분이라는 제한이 있으니까 복잡한 아이템은 빼고 조작은 터치 하나로만 끝냅시다."

💡 **이때 마음에 안 드는 부분이 있으면?**
언제든 쓰레드 채팅창에 `@디자이너 야, 장애물을 소행성 말고 외계인으로 바꿔봐` 라고 치시면 됩니다. 에이전트가 문맥을 이해하고 기획을 수정합니다.

### 🚦 Step 3: 팀장(사용자)의 결재 (위험도 평가)

회의가 끝나면 AI CEO가 기획의 위험도(Risk)를 평가합니다.

- **LOW / MEDIUM:** "무난한 기획이네요!" 15초 뒤에 자동으로 다음 단계로 넘어갑니다.
- **HIGH:** "이 기획은 규모가 너무 커서 문제가 될 수 있습니다." 경고와 함께 봇이 멈춥니다.

이때 채팅창에 2개의 버튼이 뜹니다.

- **[✅ 이대로 진행]**: "내 맘에 드니까 그냥 만들어!" (승인)
- **[✏️ 피드백 주고 수정]**: "아니, 기획안 3번 항목이 마음에 안 들어. 이렇게 저렇게 바꿔." (수정 지시)

### 🚀 Step 4: 자동 코딩 및 GitHub 자동 전송 (Handoff)

승인이 떨어지면 봇이 순식간에 **최종 기획서**, **데이터 스키마**, **실제 플레이 가능한 HTML 게임 코드**를 작성합니다.
그리고 아무도 모르게 백그라운드에서 아까 연동해 둔 여러분의 **GitHub에 브랜치를 만들고 코드를 올린 뒤 Pull Request(PR)까지 알아서 생성**합니다!

디스코드에 **"GitHub PR created: 링크"** 가 뜨면 링크를 눌러보세요.
개발팀이 바로 확인하고 작업할 수 있도록 기획 요약과 파일들이 완벽하게 정리되어 있습니다.

## 3. ⌨️ 명령어 총정리 (Commands)

| 슬래시 커맨드                    | 설명                                                                                                                                       |
| :------------------------------- | :----------------------------------------------------------------------------------------------------------------------------------------- |
| `/tutti start name:... idea:...` | 새로운 게임 기획을 시작합니다.                                                                                                             |
| `/tutti revise "지시사항"`       | 완성된 기획안이 맘에 안 들 때 엎고 새로 만듭니다.                                                                                          |
| `/tutti github`                  | GitHub 연동 가이드 메시지를 봅니다.                                                                                                        |
| `/tutti settings`                | 사용할 AI 모델이나 GitHub 레포 주소를 바꿉니다.                                                                                            |
| `/tutti apikey`                  | (DM으로) API 키나 GitHub 토큰을 등록합니다.                                                                                                |
| `/tutti learn "지시"`            | 에이전트들이 앞으로 게임을 기획할 때 절대 까먹지 않도록 "이 프로젝트의 규칙"을 가르칩니다. (예: "우리 게임은 무조건 픽셀 아트 스타일이야") |

## 4. ⚠️ 기술적 한계 및 안전망

이 봇은 거대한 시스템을 위해 **로컬 소형 AI 모델(Qwen 2.5 7B 등)**을 사용하도록 기본 세팅되어 있습니다.

소형 모델이다 보니 복잡한 게임 로직(JavaScript)을 짜거나 완벽히 동작하는 게임을 한 번에 뽑아내는 데는 잦은 버그가 발생하거나 한계가 있습니다. (고품질 CSS 템플릿을 입혀놓아도, 로직 자체가 엉성할 때가 많습니다.)

하지만 걱정하지 마세요! **이것이 바로 이 봇의 핵심 기획 의도입니다.**

1. **프로토타입은 거들 뿐:** AI가 만든 코드는 완제품이 아니라 **"내 아이디어가 이런 느낌이다"**를 보여주기 위한 뼈대(Proof of Concept)입니다.
2. **그래서 GitHub PR이 존재합니다:** 코드가 완벽하지 않기 때문에, 봇이 스스로 런칭하지 않고 **개발팀(사람)에게 "기획서랑 초안 코드 다 짰으니, 이제 제대로 수정해서 배포해 주세요" 라고 PR을 넘기는(Handoff) 구조**인 것입니다.
3. **안전망 아키텍처:** 최소한 보기에는 그럴싸하도록 미리 준비된 **고품질 CSS 템플릿**이 강제로 덮어씌워져, 개발팀이 초안을 받아볼 때 시각적인 거부감을 줄여줍니다.

_(물론 나중에 `/tutti settings`에서 OpenAI GPT-5.5나 Claude Opus 4.7 등 최신 대형 모델을 연동하면 이 한계조차 극복하고 훨씬 고퀄리티의 플레이 가능한 코드를 만들어냅니다!)_
